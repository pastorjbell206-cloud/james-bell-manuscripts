# Scripts

Tooling for moving manuscripts from this repo into the LiveWell website database.

## `import-manuscripts.mjs`

Walks each numbered category folder and inserts every `.md` manuscript into the LiveWell DB:

- `01_Major_Books`, `02_Church_Governance_Series` → `books` table
- `03_Booklets`, `04_HB_Series`, `05_PCN`, `08_Articles` → `posts` table
- `06_FBC`, `07_ResourceWall` → `resources` table
- `09_Other` → skipped by default (use `--include-other`)

Inserts use idempotent upserts on `slug`, so the script is safe to re-run.

### Why this isn't runnable from this repo alone

This script imports `./server/db.ts` and `./drizzle/schema.ts` from the LiveWell **website** repo (`pastorjbell206-cloud/livewell`). That repo is currently archived (read-only as of 2026-04-17), so:

1. **Unarchive** `pastorjbell206-cloud/livewell`, **OR**
2. **Migrate** the LiveWell source into a new repo, **OR**
3. **Clone** the archived repo locally as a working directory.

Then copy `scripts/import-manuscripts.mjs` and `scripts/manifest.json` into the LiveWell repo root and run from there.

### Running

```bash
# Preview every insert without touching the database:
node import-manuscripts.mjs --dry-run

# Real import — all categories except 09_Other:
node import-manuscripts.mjs

# Import a single folder (good for testing):
node import-manuscripts.mjs --only=05_PCN

# Include the 09_Other working files too (after you've cleaned them up):
node import-manuscripts.mjs --include-other
```

Set `MANUSCRIPTS_DIR` if this repo lives somewhere other than `../james-bell-manuscripts`:

```bash
MANUSCRIPTS_DIR=/path/to/james-bell-manuscripts node import-manuscripts.mjs --dry-run
```

### What gets inserted

For each manuscript:

| Field | Source |
|---|---|
| `title` | YAML `title_guess` → first `**bold**` line in body → cleaned filename |
| `slug` | lowercase-kebab of the title (max 200 chars) |
| `body` (posts) | full markdown body |
| `description`/`excerpt` | first paragraph >60 chars from the body |
| `readingTimeMinutes` | `word_count` from front-matter / 220 |
| `topic` (posts) | inferred from title keywords (justice / leadership / etc.) |
| `audience` (posts) | folder default, e.g. `pastors` for PCN |
| `bookType` (books) | `authored` |
| `sortOrder` (Governance Series) | `Book##` number from filename |
| `published` | `true` for most folders, `false` for FBC and 09_Other |

### Duplicates

The root `README.md` of `james-bell-manuscripts` flags ~18 duplicate / near-duplicate files. Those are listed in `DUPES_TO_SKIP` inside the script and are not imported. Update that list if you delete the canonical files instead.

### After the import

1. Spot-check a few rows in the LiveWell admin UI: `/admin/posts`, `/admin/books`, `/admin/resources`.
2. Manually rename anything where the auto-generated title is too generic (mainly 02_Church_Governance and 05_PCN).
3. Set `published = true` on FBC resources you want public.
4. Add cover images / SEO fields via the admin UI — the script doesn't touch those.
